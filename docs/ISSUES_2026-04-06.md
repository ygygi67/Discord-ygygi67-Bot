# ปัญหาและสิ่งที่ปรับปรุง (6 เม.ย. 2026)

## ปัญหาที่พบ

1. **โหลด Cog `cogs.utility.utility` ไม่สำเร็จ**
   - Error: `CommandLimitReached: maximum number of slash commands exceeded 100 globally`
   - สาเหตุ: คำสั่งกลุ่ม Utility ถูกลงทะเบียนแบบ Global มากเกินเพดานของ Discord

2. **คำสั่งไม่ครบในบางเซิร์ฟเวอร์**
   - ตัวอย่าง: เซิร์ฟเวอร์หนึ่งมี `/โหลดคลิป` แต่อีกเซิร์ฟเวอร์ไม่แสดง
   - สาเหตุ: การ sync คำสั่งฝั่ง guild ยังไม่ครอบคลุมทุก guild ที่บอทใช้งาน

3. **คำสั่ง `/sync` ล้มเหลว**
   - Error: `'DistributedAlphaBot' object has no attribute 'sync_commands'`
   - สาเหตุ: เรียกเมธอดที่ไม่มีอยู่จริงในคลาสบอท

## สิ่งที่ทำ/เพิ่มแล้ว

### 1) แก้ระบบ `/sync` ให้ใช้งานได้จริง
- เปลี่ยนการเรียกจาก `self.bot.sync_commands(...)` เป็น `self.bot.tree.sync(...)`
- เพิ่มโหมด `scope=all` เพื่อ sync ทุกเซิร์ฟเวอร์ที่บอทอยู่ในครั้งเดียว

### 2) ลดความเสี่ยงชนเพดาน 100 คำสั่ง Global
- ปรับ `Utility` ให้ลงทะเบียนแบบ **guild-scoped หลายเซิร์ฟเวอร์** เป็นค่าเริ่มต้น
- รองรับการอ่าน guild หลายแหล่ง:
  - `DISCORD_GUILD_ID`
  - `DISCORD_GUILD_IDS` (หลายค่า)
  - โฟลเดอร์ใน `data/` ที่ขึ้นต้นด้วย GID
- เพิ่มสวิตช์บังคับกลับเป็น Global ได้ด้วย `FORCE_GLOBAL_UTILITY_COMMANDS=1`

### 3) ทำให้คำสั่งกระจายครบหลายเซิร์ฟเวอร์มากขึ้น
- ตอนเริ่มบอท จะ sync global + sync แบบ guild ให้กับ **ทุก guild ที่ระบบรู้จัก**
- ลดเคสคำสั่งขึ้นไม่ครบข้ามเซิร์ฟเวอร์หลังรีสตาร์ท

## ผลลัพธ์ที่คาดหวังหลังแก้

- Cog `Utility` ควรโหลดได้โดยไม่ติด `CommandLimitReached` (ในค่าตั้งต้น)
- `/sync` ใช้งานได้ และมีโหมด sync ทุกเซิร์ฟเวอร์
- คำสั่งสำคัญ (เช่น `/โหลดคลิป`) ควรเห็นได้สม่ำเสมอมากขึ้นระหว่างหลายเซิร์ฟเวอร์

## หมายเหตุ

- หากต้องการให้ Utility เป็น Global ทั้งหมดจริง ๆ ให้ตั้ง:
  - `FORCE_GLOBAL_UTILITY_COMMANDS=1`
- แต่หาก Global commands เกิน 100 อีกครั้ง Discord จะปฏิเสธการลงทะเบียนตามข้อจำกัดแพลตฟอร์ม

## อัปเดตล่าสุด (7 เม.ย. 2026)

### ปัญหาเพิ่มเติมที่พบ

1. **ยังชนเพดาน 100 commands แม้ log บางช่วงแสดง 103 คำสั่ง**
   - สาเหตุ: ตัวเลขใน log คือรายการที่บอทรวบรวมจาก `global + guild` เพื่อแสดงผล ไม่ใช่จำนวนที่ Discord ยอมรับแบบ global ล้วน
   - ผลกระทบ: Cog บางตัว (โดยเฉพาะ Utility) อาจโหลดไม่สำเร็จและทำให้คำสั่งหาย

2. **บางเซิร์ฟเวอร์ได้ `0 guild commands`**
   - สาเหตุ: Cog ถูกลงทะเบียนแบบ guild-scoped เฉพาะบาง guild (เช่นพบแค่ `DISCORD_GUILD_ID` เดียว)
   - ผลกระทบ: เซิร์ฟที่ไม่ได้ถูกผูก scope จะไม่เห็นคำสั่งหมวดนั้น

3. **sync ล้มเหลวด้วย `NoneType can't be used in await expression`**
   - สาเหตุ: มีการ `await` เมธอด `clear_commands(...)` ซึ่งไม่ใช่ async
   - ผลกระทบ: `/sync` ทำงานไม่จบ และเกิด `CommandNotFound` ต่อเนื่อง

### สิ่งที่ปรับเพิ่ม

- แก้ `/sync` ให้ใช้ `clear_commands(...)` แบบไม่ `await`
- เพิ่มการล้างคำสั่งทุก guild เมื่อใช้ `scope=all` + `force=true`
- เพิ่มโหมดกรองคำสั่ง AI และ Utility ตั้งแต่ขั้นตอนโหลด cog เพื่อลดคำสั่งย่อย
  - `AI_DISABLE_SLASH=1` (ปิด slash AI ทั้งหมด)
  - `AI_PANEL_ONLY=1` (เปิดเฉพาะ `/ai` เมื่อไม่ปิดทั้งหมด)
  - `UTILITY_MAIN_ONLY=1` (คงเฉพาะคำสั่งหลักของ Utility)
  - `UTILITY_CORE_COMMANDS` (กำหนดรายชื่อคำสั่งหลักได้)

### แนวทางใช้งานที่แนะนำ (ให้คำสั่งเหมือนกันทุกเซิร์ฟ และไม่เกินเพดาน)

1. ลด global commands จริง โดยตัดคำสั่งย่อยหนัก ๆ (AI/Utility)
2. ทำคำสั่งหลักให้ global เท่านั้น
3. รีซิงก์ใหม่แบบล้างของค้าง:
   - `/sync scope: all force: true`
   - `/sync scope: global force: true`
   - `/sync scope: all force: false`
